# Создатель шортсов

Windows-приложение на Java для нарезки YouTube Shorts из локальных видеофайлов, включая `.mkv`.

Приложение использует FFmpeg для работы с медиа и эвристический анализ, чтобы находить интересные моменты:

- читает локальные видеофайлы `.mkv`, `.mp4`, `.mov`, `.webm`, `.avi`;
- анализирует пики громкости;
- анализирует движение и контраст в кадре в Java-режиме;
- предлагает непересекающиеся фрагменты между выбранной минимальной и максимальной длительностью;
- позволяет выбрать количество шортсов или автоматически найти все подходящие моменты;
- позволяет выбирать, прослушивать, отключать, смешивать и настраивать громкость аудиодорожек;
- умеет останавливать долгий анализ или экспорт кнопкой `Отмена`;
- поддерживает русский и английский язык;
- может опционально использовать `analysis\analyze.py` для Python-анализа с автоматическим fallback на Java;
- создаёт готовые промпты для заголовков/превью и несколько 16:9 кадров, которые можно вручную приложить в ChatGPT;
- выбирает более активную область кадра для вертикального кропа, а не всегда центр;
- экспортирует вертикальные `.mp4` 1080x1920 для YouTube Shorts;
- сохраняет шортсы в папку с названием исходного видеофайла.

## Требования

- Java 17 или новее
- FFmpeg в `PATH` или в папке `ffmpeg\bin` рядом с приложением
  - `ffmpeg`
  - `ffprobe`
  - `ffplay`
- Python нужен только для режима анализа через Python. Это необязательно.

## Полный запуск по шагам

1. Установи Java 17 или новее.

   Проверь в PowerShell:

   ```powershell
   java -version
   javac -version
   ```

   Должны работать обе команды. Приложение компилируется при запуске, поэтому нужен именно JDK с `javac`, а не только обычная Java Runtime.

2. Установи FFmpeg.

   Из папки проекта запусти:

   ```powershell
   .\install-ffmpeg.bat
   ```

   В PowerShell важно писать именно `.\install-ffmpeg.bat`, потому что команды из текущей папки не запускаются без `.\`.

3. После установки FFmpeg закрой и снова открой PowerShell.

   Так Windows обновит `PATH`. Приложение умеет искать FFmpeg и в типичных папках WinGet, но перезапуск PowerShell помогает избежать путаницы.

4. Перейди в папку проекта.

   Например:

   ```powershell
   cd "D:\projects\shorts creator"
   ```

5. Запусти приложение.

   Обычный запуск с окном консоли:

   ```powershell
   .\run.bat
   ```

   Запуск без видимой командной строки:

   ```text
   ShortsCreator.vbs
   ```

   Можно просто дважды кликнуть по `ShortsCreator.vbs`.

6. Выбери видео.

   Нажми `Обзор` рядом с полем `Видео` и выбери локальный файл `.mkv`, `.mp4`, `.mov`, `.webm` или `.avi`.

7. Проверь папку сохранения.

   Приложение автоматически предложит папку с названием исходного видеофайла.

8. Выбери аудиодорожки.

   Отметь дорожки, которые нужно использовать. Если выбрано несколько дорожек, в итоговом шортсе они будут сведены в одну. Громкость каждой дорожки можно настроить ползунком.

9. При необходимости прослушай микс.

   Нажми `Прослушать микс`, чтобы услышать выбранные дорожки вместе. Нажми `Стоп`, чтобы остановить воспроизведение.

10. Настрой параметры шортсов.

   Укажи:

   - количество шортсов или `Все моменты`;
   - минимальную длительность;
   - максимальную длительность;
   - движок анализа: `Авто`, `Java` или `Python`.

   Также заполни:

   - `Название игры`;
   - `# части`;
   - тип: обычная часть или стрим.

11. Нажми `Анализ`.

   В логе будет видно, какой режим реально используется:

   ```text
   Анализ через Java
   ```

   или:

   ```text
   Анализ через Python
   ```

12. Проверь найденные моменты.

   Выбери те фрагменты, которые хочешь экспортировать. По умолчанию выделены все найденные моменты.

13. Нажми `Экспорт`.

   Приложение сохранит отдельные вертикальные `.mp4` 1080x1920 в папку вывода.

14. При необходимости создай промпты для заголовков и превью.

   Выдели один или несколько найденных моментов и нажми `Заголовок + превью`.

   Для каждого выбранного момента приложение сохранит:

   - три кадра 16:9;
   - готовый текст промпта для ChatGPT с инструкцией для заголовков и превью.

   Промпт просит ChatGPT придумать 20 русских заголовков по приложенным кадрам и событиям в них, в формате:

   ```text
   Название игры — #части | заголовок
   Название игры — стрим #части | заголовок
   ```

   Файлы появятся здесь:

   ```text
   <папка вывода>\thumbnail_prompts\short_XX
   ```

   После этого приложи в ChatGPT сохранённые кадры и фото кошки, затем вставь готовый промпт.

15. Если анализ, экспорт или подготовка промптов затянулись, нажми `Отмена`.

   Это остановит текущую работу без закрытия приложения.

## FFmpeg

Для установки FFmpeg на Windows можно использовать:

```powershell
.\install-ffmpeg.bat
```

Если устанавливаешь FFmpeg вручную, добавь папку `bin` в `PATH` или распакуй FFmpeg в локальную папку `ffmpeg`.

Приложение ищет FFmpeg в таких вариантах:

```text
ffmpeg\bin\ffmpeg.exe
ffmpeg\ffmpeg-...\bin\ffmpeg.exe
tools\ffmpeg\bin\ffmpeg.exe
tools\ffmpeg\ffmpeg-...\bin\ffmpeg.exe
```

## Проверка сборки

Чтобы только проверить, что приложение компилируется:

```powershell
.\run.bat --build-only
```

Если всё хорошо, будет:

```text
Build succeeded.
```

## Режимы анализа

- `Авто`: сначала пробует Python-анализ, если не получилось - использует Java.
- `Java`: всегда использует встроенный Java-анализ.
- `Python`: пробует Python-анализ, но при проблеме всё равно откатывается на Java.

Python-режим сейчас быстрее, потому что анализирует звук легче и не делает полный визуальный анализ. Java-режим медленнее, зато учитывает не только звук, но и движение/контраст в кадре.

## Примечания

Автоматический поиск интересных моментов - эвристический. Смех, реакции, крики, музыкальные акценты, активное движение, смены сцен и визуально насыщенные фрагменты обычно получают более высокий score.

Если выбрать конкретное количество шортсов, приложение берёт лучшие непересекающиеся варианты. Если выбрать `Все моменты`, приложение всё равно ищет локальные пики интересности и не должно просто нарезать всё видео подряд.

Если в видео несколько аудиодорожек, первая выбирается по умолчанию. Можно выбрать одну дорожку или несколько. Несколько выбранных дорожек будут сведены в одну AAC-дорожку при экспорте. Ползунки громкости влияют и на предпрослушивание, и на итоговый экспорт.
